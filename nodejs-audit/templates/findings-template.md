# Finding template

Используй этот шаблон в отчётах для каждой найденной проблемы.

```markdown
### <ID>: <короткое название>

**Категория:** <Architecture / Security / Performance / Readability / ...>
**Серьёзность:** <Critical / High / Medium / Low / Info>
**Файл(ы):** src/X.ts:45, src/Y.ts:12
**Уверенность:** <High / Medium / Low>

**Что:** [одно предложение, что не так]

**Почему это плохо:** [последствия — что может произойти]

**Текущий код:**
\```typescript
// src/X.ts:45
const result = data.map(item => fetchDetails(item.id));
\```

**Как должно быть:**
\```typescript
const result = await Promise.all(data.map(item => fetchDetails(item.id)));
\```

**Усилия на фикс:** [Низкие / Средние / Высокие]
**Зависимости:** [есть ли блокеры — другие задачи, которые надо сделать сначала]

**Готовый промт для ИИ:**
\```
[самодостаточный текст промта, который можно скопировать в Claude Code]
\```
```

## Уровни серьёзности — как калибровать

- **Critical** — продакшн-инцидент возможен сегодня. Утечка данных, RCE,
  выход из строя сервиса под нагрузкой 1 запрос.
- **High** — проблема, которая вылезет в течение недели после деплоя.
  Падение под малой нагрузкой, серьёзная регрессия, обход авторизации
  при определённых условиях.
- **Medium** — технический долг или баг с малой вероятностью. Стоит
  исправить в ближайший месяц.
- **Low** — улучшение, которое заметит только разработчик.
- **Info** — наблюдение, не баг. "Можно сделать ещё лучше, но и так норм".

## Уровни уверенности — как калибровать

- **High** — я воспроизвёл проблему / есть конкретный код, который её
  показывает.
- **Medium** — паттерн похож на проблемный, но без воспроизведения.
- **Low** — гипотеза. Нужна ручная проверка.

При Low уверенности и Critical серьёзности — пометь как "требует
ручной проверки" и не приоритизируй выше реальных проблем.
